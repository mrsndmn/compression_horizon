# Habr article — Progressive Cramming (ICML 2026)

Draft of a popular-science article for [habr.ru](https://habr.com) that does a
deep dive into the main results of *"Progressive Cramming: Reliable Token
Compression and What It Reveals"* (ICML 2026). Written in Russian, hook-first,
in the AIRI company-blog style.

## Layout

- `habr.md` — the article draft (Russian, ~3200 words). Embeds figures/animations
  from `assets/` with relative paths; numeric results are native Markdown tables.
- `assets/` — figures and animations, copied from the poster (`../poster/images/`)
  and the public repo (`FusionBrainLab/progressive_cramming`, `presentation/assets`
  and `page/assets`). The `.mp4` animations are meant to be embedded as Habr video
  (or re-uploaded to habrastorage) on publication.
- `refs/` — reference articles scraped from Habr for style calibration
  (2 AIRI posts + the Kuratov mem-vectors review). Generated by the scraper below;
  **not** part of the article.

## Regenerating the scraped references

```bash
python scripts/habr/scrape_habr.py            # default set: 906592, 804515, 816125
python scripts/habr/scrape_habr.py <id|url>   # scrape additional articles
python scripts/habr/scrape_habr.py --force    # re-scrape existing
```

The scraper reuses the approach of Ilya Gusev's
[`create_habr.py`](https://github.com/IlyaGusev/rulm/blob/master/data_processing/create_habr.py):
it queries the public Habr API v2 and converts the article HTML to Markdown
(via BeautifulSoup, since `html2text` is not installed here).
